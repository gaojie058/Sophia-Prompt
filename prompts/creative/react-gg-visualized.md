# React Visualized — Design Style Reference

**Source:** https://react.gg/visualized

## Overview

React.gg's "Visualized" page is an interactive, scroll-driven visual exploration of core React concepts. It transforms abstract programming ideas into engaging animated illustrations.

## Design Characteristics

- **Scroll-driven storytelling**: Content unfolds as the user scrolls, with animations triggered by scroll position
- **Concept visualization**: Abstract technical concepts (state, rendering, components) are represented through animated diagrams and illustrations
- **Dark theme with vibrant accents**: Clean dark background with colorful, high-contrast visual elements
- **Progressive disclosure**: Complex topics broken into digestible sections, revealed step by step
- **Sidebar navigation**: Table of contents that tracks reading progress
- **Minimal text, maximum visual**: Prioritizes animations and diagrams over walls of text
- **Historical narrative arc**: Starts with context (jQuery → Backbone → Angular) before introducing the main subject

## Key Sections & Visual Patterns

1. **History of the Web** — Timeline-style progression (jQuery → Backbone → Angular → React)
2. **The React Way** — `v = fn(s)` visualized with animated function diagrams
3. **Components & JSX** — Composition illustrated through nesting visuals
4. **State & Props** — Data flow animated with arrows and color-coded nodes
5. **Rendering** — Process diagrams showing re-render triggers and Virtual DOM diffing
6. **Effects & Refs** — Lifecycle visualized as timelines
7. **Concurrent Rendering** — Before/after comparisons (React 17 vs 18)
8. **Server Components** — Client/server boundary illustrated

## When to Use This Style

- Educational/explainer content for technical topics
- Product feature walkthroughs
- Documentation that needs to be engaging, not just informative
- Landing pages that tell a story through scroll interaction
